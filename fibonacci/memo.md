# 自分が書いたコードの問題点
## 無駄なメモリ消費:
再帰的に呼び出す度に、スタックに情報が積まれます。これにより、n が大きくなるとスタックオーバーフローやメモリの無駄遣いが生じることがあります。
また、再帰関数は結果を保存せずに繰り返し計算しているため、冗長な計算が発生し、パフォーマンスが低下します。

## 再帰的なメソッド呼び出しによるパフォーマンス問題:
再帰的な呼び出しが続くと、計算結果を再度計算し直すことになります。特に n が大きくなると、この「重複計算」が増え、効率が悪化します。

# ChatGPTのコードの良い点：
## メモ化（メモリ内での結果の保存）:
計算した結果を memo に保存し、次回同じ値が必要になった場合には再計算せずにメモリから取得することで、無駄な計算を省略しています。これにより、計算量が減り、効率的に動作します。

## 非再帰的アプローチ:
memo を使って反復処理を行うことで、再帰的なメソッド呼び出しを避けています。これにより、スタックオーバーフローのリスクがなく、メモリの使用量も抑えられます。
時間計算量の改善:

計算量は O(n) となり、冗長な計算が発生しません。再帰を使わずに、必要な値だけを順番に計算していくため、n が大きくなっても効率的に動作します。
